What is SignalServer?

'Splat!" is a simulation, originally created by John Magliacane (KD2BD), which can be used to model or predict signal coverage. Given a set of parameters which describe the transmitter, the intended receiver, and their respective antenna characteristics, Signal Server creates a prediction that takes such variables as terrain features, propagation characteristics, power levels, receiver sensitivities, and other factors into account.

Signal Server GUI

Signal Server GUI is designed as a companion to Signal Server. Signal Server GUI provides a simple and intuitive interface to manage the many configuration options available within Signal Server. This web interface allows you to build and save three item types (Stations, Antennas, and Plots). As you develop plots you are afforded opportunity to manipulate some of the many parameters which can affect propagation. Reasonable default values are provided, which can be changed as you develop and refine your model.

Plots

When generated, plots create a variety of files. Some of the possible outputs are listed below.

Generated Files:
Basic plot output:
  • .zip - A compressed zip file containing all generated files.
  • .kmz - A Keyhole Markup Language Zip file ready for import into Google Earth.
  • .png - A Portable Network Graphic image file.
  • .ppm - A Portable Pixel Map image file.
Additional file from point-to-point analysis:
  • _curvature - A compressed zip file containing all generated files.
  • _fresnel
  • _fresnel60
  • _profile
  • _reference
  • .txt - A text based report of the point-to-point analysis.
  • _ppa.png - A Portable Network Graphic image of the point-to-point analysis.

Caveat

SignalServer, like all simulation programs, is at best an approximation of reality. A great many things can disturb the accuracy of the results. Even an inaccurate simulation may be able to provide some guidance to a real world situation, but there is no guarantee that the output is useful for anything. Results provided by this website may or may not be applicable to the 'real world', and should always be interpreted with a large measure of skepticism.
